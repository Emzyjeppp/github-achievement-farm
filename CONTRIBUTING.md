# Contributing to DevOps Stress Testing Sandbox

First off, thank you for considering contributing to this sandbox! This repository is primarily a testing ground for Git CLI operations, CI/CD Actions, and API rate-limiting behaviors.

## How to Contribute

1. **Fork the Repository:** Create a personal copy of this repository on your GitHub account.
2. **Create a Branch:** Create a branch for your experiments (e.g., `git checkout -b feature/new-benchmark`).
3. **Make Changes:** Add new shell scripts or configuration variables inside the `scripts/` or `config/` directories.
4. **Submit a Pull Request:** Open a PR back to the main branch of this repository.

## Development Standards
- Ensure all shell scripts are linted and formatted.
- Do not check in active personal access tokens (PATs) or sensitive keys.
